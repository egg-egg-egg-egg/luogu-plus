**洛谷已迁移到新前端架构 "Spilopelia"**：
- ❌ 旧的 `window.__INITIAL_STATE__` 已不存在

- AC 列表路径：`data.passed`（非 `passedProblems`）
- 每条字段：`{type, name, difficulty, pid}`

### 字段映射
| 我们的模型 | 洛谷源字段 | 路径 |
|---|---|---|
| problemId | `pid` | `data.passed[i].pid`（字符串，如 "P1001"/"B2001"） |
| title | `name` | `data.passed[i].name`（已含 [NOIP ...] 前缀） |
| difficulty | `difficulty` | `data.passed[i].difficulty`（number 0-7） |
| type | `type` | `data.passed[i].type`（"P"/"B"，历史可能有 T） |

### 一致性校验
`len(data.passed) === data.user.passedProblemCount`（实测 206=206）

### 提取正则
```python
m = re.search(r'<script id="lentille-context" type="application/json">(.*?)</script>', html, re.DOTALL)
state = json.loads(m.group(1))  # 直接 JSON，无 URL decode
```

### 风险
- 架构可能再次变更 → 提取器双路兼容（lentille-context 优先，__INITIAL_STATE__ 回退）
- `data.passed[i]` 不含题目标签、AC 时间、提交 ID
- `data.elo`、`user.elo/eloValue/badge/slogan` 等字段可空，需做空值保护

## ② 频率限制实测

### 测试方案
- 未登录、无 cookie、真实浏览器 UA
- 第一轮：2s 间隔 × 20 次
- 第二轮：3s 间隔 × 20 次（第一轮后等 5 分钟）
- 目标：`https://www.luogu.com.cn/user/1403058/practice`

### 实测结果

| 轮次 | 间隔 | 触发风控 | 状态码 | 响应大小 | 耗时范围 |
|---|---|---|---|---|---|
| 第一轮 | 2s | **否** | 全部 200 | 24228 字节稳定 | 0.277-0.357s |
| 第二轮 | 3s | **否** | 全部 200 | 24228 字节稳定 | 0.284-0.491s |

### 结论
- **2s 间隔连发 20 次完全安全**，无风控、无退化
- **推荐默认间隔：2 秒**
- 3s 作为保守降级档备用
- 5s 无必要，过于保守

### 风控特征（未观测到，预期）
- 状态码：403 / 429 / 503
- 关键词："验证"/"频繁"/"block"
- 响应大小偏离基线 >30%
- 重定向到验证码页

### 恢复策略
- 指数退避：30s → 60s → 120s，最多 3 次
- 触发后冷却 5 分钟（C3VK cookie 有效期 5 分钟）

## ③ MV3 SW 保活原型

### 选定方案
chrome.alarms 24s 心跳 + chrome.storage.local 断点续传 + SW 重启顶层 bootstrap = 三重保险

### 原型架构
```
popup.js ──START_TASK──→ background.js
                           ├── 创建 alarm (24s 周期)
                           ├── 启动 40s 长任务 (20 × 2s)
                           ├── 每 tick 写 storage.local 进度
                           ├── alarm 触发 → getPlatformInfo() 保活
                           ├── 任务完成 → 清除 alarm
                           └── SW 重启 → 顶层读 storage.local 续传

popup.js ──GET_STATUS──→ background.js ──→ 返回 {done, total, status}
popup.js ──RESET──→ background.js ──→ 清除任务记录
```

### 核心代码片段
```js
// 保活：24s alarm < 30s 休眠阈值
chrome.alarms.create('keepalive', { periodInMinutes: 0.4 });
chrome.alarms.onAlarm.addListener(() => {
  chrome.runtime.getPlatformInfo(() => {});  // 重置空闲计时器
});

// 断点续传：每 tick 写进度
chrome.storage.local.set({ task: { done, total, status: 'running' } });

// SW 重启恢复：顶层代码
chrome.storage.local.get('task', ({ task }) => {
  if (task?.status === 'running') resumeFrom(task.done);
});
```

### 手动验证步骤（待用户执行）
1. chrome://extensions 加载已解压扩展 `预研/sw-keepalive-prototype/`
2. 点扩展图标 → 启动任务 → 立即关闭 popup

5. 断点续传：启动任务 → 手动 kill SW → 重新打开 popup → 应从断点继续

### 代码复用
原型 → 主项目 `src/background/` 迁移映射：
- `background.js` → `src/background/index.ts`（保活+消息）
- 任务循环 → `src/background/batch-task.ts`（tick 内换成真实 fetchStudentAC）
